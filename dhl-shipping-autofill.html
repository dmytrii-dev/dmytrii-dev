Shipping workflow

DHL Shipping Autofill.

A browser extension workflow that turns pasted receiver details into prepared DHL shipping fields.

Problem

Receiver address details had to be copied from text into shipping form fields by hand.

System

The extension parses the pasted address and prepares matching receiver-field values.

Control

The user can check the parsed result before using it in the DHL form.

01 · Ready state

Start from a clear input view.

The workflow opens with a visible receiver-address area, an empty preview section, and two clear actions: preview first or fill later.

02 · Address pasted

The address block stays visible before any fill action.

The copied receiver block can be reviewed inside the extension first. Nothing is pushed into the carrier page yet, so the user keeps control over the next step.

03 · Parsed result

Preview the separated fields before using them.

The extension prepares the receiver name, company, street, number, postal code, city, and country. The user can check the parsed structure before continuing in the DHL workflow.

Project scope

What this page shows.

Public sample data. Visible names, addresses, and form values are examples for demonstration.

Client details stay private. Account access, internal rules, and real shipment data are not included.

Independent workflow. DHL is referenced only as the carrier website this workflow was built around. This project is independent and not affiliated with DHL.
